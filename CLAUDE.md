# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Install dependencies (includes @anthropic-ai/sdk and tsx)
npm install

# Install Playwright browsers
npx playwright install --with-deps

# Run all tests
npx playwright test

# Run a specific test file
npx playwright test seed.spec.ts

# Run tests in headed mode (visible browser)
npx playwright test --headed

# Run tests in a specific browser
npx playwright test --project=chromium
npx playwright test --project=firefox
npx playwright test --project=webkit

# Debug a test
npx playwright test --debug

# View test report
npx playwright show-report
```

## Running the Orchestrator

```bash
# Set your Claude API key
export ANTHROPIC_API_KEY="sk-ant-..."

# Run the full pipeline (public app, no login)
npx tsx orchestrate.ts --url https://yourapp.com

# Run with credentials (handles MFA/OTP interactively)
npx tsx orchestrate.ts --url https://yourapp.com --username you@co.com --password secret

# Or via environment variables
TEST_URL=https://yourapp.com TEST_USERNAME=you@co.com TEST_PASSWORD=secret npm run orchestrate
```

The orchestrator opens a **visible browser window** so you can see what is happening and respond to OTP prompts in your terminal.

## Architecture

This is a Playwright end-to-end test automation project with an **AI-assisted test development workflow** using GitHub Copilot agents.

### AI Agent Workflow (`.github/agents/`)

Three custom agents form a pipeline for test development:

1. **`playwright-orchestrator.agent.md`** ⭐ — **Entry point for the full pipeline.** Accepts a URL + optional credentials, coordinates all phases: explore → plan → await user approval → generate tests → heal failures. Invoke with: `"URL: https://example.com | Username: admin | Password: secret"`
2. **`playwright-test-planner.agent.md`** — Explores a live web app (supports credentials for login) and produces a structured test plan saved to `specs/`
3. **`playwright-test-generator.agent.md`** — Reads a test plan from `specs/` and generates executable `.spec.ts` test files
4. **`playwright-test-healer.agent.md`** — Runs failing tests, analyzes errors, and patches selectors/assertions to fix them

All agents communicate with Playwright via the MCP server configured in `.vscode/mcp.json` (`npx playwright run-test-mcp-server`).

### Key Files

- `seed.spec.ts` — Starter test template; copy and adapt for new test suites
- `specs/` — Test plan documents (markdown); input for the generator agent
- `specs/saucedemo-test-plan.md` — Test plan for www.saucedemo.com (e-commerce demo)
- `specs/cloudqa-comprehensive-test-plan.md` — Test plan for https://inbox.cloudqa.io/ (email platform)
- `test-results/` — Generated by Playwright after test runs (screenshots, traces, videos)

### Test Structure Convention

Tests follow the `@playwright/test` pattern:

```typescript
import { test, expect } from "@playwright/test";

test.describe("Suite Name", () => {
  test("test case name", async ({ page }) => {
    await page.goto("https://example.com");
    // ...
  });
});
```

### `playwright.config.ts`

Runs all `**/*.spec.ts` files across Chromium, Firefox, WebKit. Workers = 1 (sequential). Also defines a `setup` project that runs `auth.setup.ts` to refresh saved auth state.

### Authentication & MFA Strategy

`auth.setup.ts` handles the one-time login (including OTP prompts) and saves the session to `auth/auth-state.json`. Tests that declare `test.use({ storageState: 'auth/auth-state.json' })` skip login entirely — MFA is never re-triggered during test runs.

```bash
# First time or when session expires — handles OTP interactively:
TEST_URL=https://yourapp.com TEST_USERNAME=you@co.com TEST_PASSWORD=secret npx playwright test --project=setup

# Run all tests (session must already be saved):
TEST_USERNAME=you@co.com TEST_PASSWORD=secret npx playwright test
```

`auth/auth-state.json` is gitignored — never committed.
